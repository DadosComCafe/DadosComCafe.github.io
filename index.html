  • Consumindo api do Starwar, e modelando com pydantic

    O Pydantic é uma ferramenta que é usada para a validação de dados. Como no python, por padrão não se declara o tipo das variávies, o pydantic pode resolver esta questão.
    Então, obtendo os valores da api do starwars, através da biblioteca requests, eu os recebo, e os aplico a models. Desta forma, cria-se um objeto pydantic para cada valor que é recebido da api. Então, declara-se a forma dos dados como classes com atributos. A vantagem, do objeto pydantic é a garantia de que ele vai corresponder a tipagem esperada, do contrário, vai quebrar. Quanto antes quebrar um dado errado, melhor! Outra coisa bastante interessante que é apresentada neste repositório, é a facilidade de implementar herança entre os objetos pydantic, o que faz total sentido dependendo do tipo de dados.
    O repositório pode ser acessado aqui: swapi_study


  • ETL marota com Airflow, GCP

    Neste projeto, desenvolvo um ETL, e crio bancos de dados com o docker: -se inicia com uma task que faz download do dataset, onde o nome do dataset o do author são consultados no mongo, então os arquivos do zip baixado são enviados para o Storage, para ser usado como DataLake. Os dados do csv são "tratados", e estes dados, agora com tipos específicos, já que inicialmente são todos textos, são então enviados para um banco de dados postgres local. Estes dados do postgres, são então migrados para o BigQuery. A ideia deste projeto é de entender melhor algumas features do airflow, e também os produtos usados, da GCP. O repositório pode ser acessado aqui:Airflow ETL


  • Download massivo de comentários

    Neste projeto, utilizando a api do youtube, faz-se um get request para obter todo o conteúdo dos comentários, threads e likes de um vídeo específico. Então a ideia é, criar um datalake com o AWS S3 (onde o json de cada request será armazenado), utilizar o pydantic para "modular" e ficar mais fácil a manipulação destes dados, aplicar alguma lógica que consiga extrair uma análise de sentimentos, e utilizar o RedShift como lugar onde construir o data warehouse. Projeto em fase inicial. O repositório pode ser acessado aqui:Airflow Youtube API create datalake
